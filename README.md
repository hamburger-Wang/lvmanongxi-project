# lvmanongxi-project
轻量多尺度遥感特征融合的作物智能识别系统

## 项目概述
### 1.1 项目背景
随着精准农业的发展，卫星/无人机遥感数据的规模化应用成为农业生产决策的核心支撑。本项目面向农业生产场景，开发集**数据导入、作物分类、长势预测、种植建议**于一体的可视化桌面应用，解决传统农业分析流程繁琐、模型调用门槛高的问题，为农业从业者提供高效、易用的智能分析工具。

### 1.2 核心目标
- 降低农业AI模型使用门槛，通过可视化界面实现无代码化操作
- 支持多源农业数据（卫星/无人机/融合数据）的批量导入与解析
- 集成作物分类与长势预测模型，输出可落地的种植建议
- 保证界面流畅性，通过多线程实现模型异步运行，避免卡顿

### 1.3 适用场景
- 农业科研机构：作物生长态势分析、模型效果验证
- 农业生产企业：规模化农田的产量预测、精准管理决策
- 大创/毕设项目：农业AI+可视化方向的完整工程实践

### 1.4文件介绍

main.py	                     
程序唯一入口，启动应用、配置全局字体，创建并显示主窗口

ui_main.py	                 
主窗口核心，整合登录 / 主界面，实现侧边栏模块切换、菜单栏功能

ui_login.py	                 
登录窗口，支持跳过登录，处理登录状态切换

config.py	                   
全局配置中心，管理字体、颜色、文件路径、支持的格式等常量

crop_model.py	               
作物分类 / 预测核心算法脚本，接收参数执行模型训练 / 推理

ui_modules/map_widget.py	   
地图分析模块 UI，预留卫星 / 无人机地图可视化接口

ui_modules/crop_widget.py	   
作物预测核心 UI，实现数据导入、参数配置、模型调用、结果展示

ui_modules/advice_widget.py	 
种植建议模块 UI，预留 AI 建议生成接口

core/data_importer.py	       
数据导入工具，解析多格式文件、获取文件详情、校验数据有效性

core/crop_worker.py	         
模型异步线程，调用 crop_model.py，反馈日志 / 进度 / 结果，避免界面卡顿

core/loading_thread.py	     
通用加载线程，处理数据加载进度，复用在各类需要进度展示的场景


## 技术架构
### 2.1 技术栈
| 类别         | 技术/工具                          | 说明                                  |
|--------------|------------------------------------|---------------------------------------|
| 开发语言     | Python 3.9+                        | 主开发语言，兼顾易用性与生态          |
| UI框架       | PySide6（Qt6）                     | 跨平台桌面应用开发，原生界面体验      |
| 环境管理     | Anaconda                          | 虚拟环境隔离，避免依赖冲突            |
| 核心功能     | 多线程（QThread）、子进程调用      | 异步执行模型，保证界面响应            |
| 数据处理     | Pandas/Numpy/OpenCV                | 农业数据解析、图像预处理（可选）      |
| 模型框架     | TensorFlow/PyTorch（预留）         | 作物分类/长势预测模型集成            |
| 可视化       | Matplotlib/QtCharts（预留）        | 长势趋势图表展示                      |
| 数据存储     | SQLite（可选）                     | 导入文件、预测结果持久化              |
